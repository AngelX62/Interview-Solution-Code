How far were you able to get with the exercise?
I completed a working calculator web application with the core arithmetic functionality working correctly. The main area I would still improve is the UI polish.

What challenges did you encounter in the process?
The biggest challenge was making the interface feel minimal and modern instead of just functional. The logic came together faster than the visual design.

If you were given unlimited time, what additional functionality would you include?
I would add keyboard support, calculation history, stronger error handling, and better responsive behavior across screen sizes. I would also spend more time on accessibility and visual polish.

If you used AI, please include all of your prompts.
Prompt 1: “Build a calculator web application. It should include a frontend piece and any backend logic needed to perform the calculations. Every functionality must work, and the UI must be minimal and modern.”
Prompt 2: “Improve the calculator logic and make the code cleaner and easier to maintain.”
Prompt 3: “Refine the UI so it feels more modern, minimal, and visually polished, with better spacing and alignment.”

What did the AI do well?
It did well on the functional logic and helped me get to a working version quickly. It was especially useful for structuring the calculator behavior and speeding up iteration.

What did the AI do poorly?
It struggled more with the UI and gave results that felt too generic at first. The design worked, but it did not initially match the level of polish I wanted.

For the places it did poorly, how did you change your approach to your prompts to improve its output?
I became more specific and described the design goals more clearly instead of relying on broad words like “modern” or “minimal.” Once I gave tighter direction around spacing, alignment, and visual polish, the output improved.
